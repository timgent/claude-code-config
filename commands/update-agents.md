Update the appropriate AGENTS.md file(s) with learnings from the current session that would have been helpful to know upfront.

## Goal

Review the current conversation and extract genuinely useful, reusable knowledge that belongs in an AGENTS.md file. The bar is high: only add things that would meaningfully help a future agent avoid confusion, do the right thing faster, or avoid a mistake they'd otherwise make.

## Process

### 1. Analyse the session

Wrap your analysis in `<analysis>` tags. Go through the conversation and identify:

- Mistakes or misunderstandings that happened because of missing context
- Non-obvious project conventions, constraints, or patterns discovered during the session
- Commands, workflows, or tools that aren't obvious from the code alone
- Gotchas, footguns, or things that required clarification
- Architectural decisions or reasons why things are done a certain way

For each candidate, ask:
- Would a capable agent reasonably not know this without being told?
- Would knowing this in advance have saved time or prevented an error?
- Is this stable and likely to remain true in future sessions?
- Is this specific enough to be actionable?

If you cannot identify anything that clears this bar, **stop here** and tell the user honestly that there's nothing worth adding from this session.

### 2. Find the right AGENTS.md file(s)

Learnings should go in the most specific AGENTS.md that applies:
- If the learning is about a specific subdirectory or service, find or look for an AGENTS.md in that directory
- If it's project-wide, use the root AGENTS.md
- If no AGENTS.md exists in the relevant location, note this to the user but do not create one — suggest they run `/init-agents` first

### 3. Propose the additions

Before editing anything, present the proposed additions to the user:
- Show which file each addition would go into
- Show the exact text you'd add
- Explain briefly why each one clears the bar

Use `AskUserQuestion` to ask: "Should I add these to AGENTS.md?" with options Yes / Let me review each one / No, nothing worth adding.

If the user selects "Let me review each one", go through each addition individually with AskUserQuestion.

### 4. Apply approved additions

For each approved addition:
- Read the current AGENTS.md content first
- Add the content to the most relevant section, or create a new section if needed
- Keep additions concise and actionable — avoid padding
- Do not duplicate information already present

### 5. Confirm

Tell the user what was added and to which file(s). If nothing was added (either because nothing cleared the bar or the user declined), say so clearly.

## Standards for what belongs in AGENTS.md

**Good candidates:**
- "Tests must be run with `X` flag because of `Y`"
- "Do not edit files in `X/` directly — they are generated by `Y`"
- "The `foo` function is deprecated; use `bar` instead"
- "This service requires env var `X` to be set locally"
- "PRs must target the `develop` branch, not `main`"
- "The linter config requires `X` style; the formatter does not enforce this automatically"

**Not worth adding:**
- Things that are obvious from reading the code or README
- One-off decisions specific to this session's task
- General best practices that apply to any project
- Anything speculative or unverified
- Trivia that doesn't affect how an agent would behave
